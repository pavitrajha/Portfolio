Pavitra Jha

Pavitra Jha is an offensive security researcher focused on deep system level vulnerability research across browser engines, Android internals, and Linux kernel subsystems. His work targets real-world exploitability conditions in production software rather than isolated theoretical bugs.

He is 17 years old and has been actively engaged in security research since the age of 9. His focus spans execution engine internals, kernel attack surfaces, sandbox isolation boundaries, and architecture level failure conditions that lead to privilege escalation.

Technical Capability

Browser Security - Deep analysis of V8 and JIT compilation pipelines including TurboFan optimization stages, type inference failures, memory layout manipulation, and exploitation primitives such as out-of-bounds access, type confusion, and use-after-free conditions under modern mitigation systems.
Android Security - Security analysis of Android framework and kernel boundary interactions including Binder IPC model, SELinux enforcement logic, system service attack surfaces, permission boundary violations, and kernel-user space escalation paths.
Linux Kernel Security - Kernel level vulnerability research across drivers, memory management, concurrency control, and race condition exploitation with focus on unsafe memory access patterns and upstream remediation behavior in production kernels.
Sandbox & Mitigations - Analysis of ASLR, KASLR, CFG, CFI, PAC, and sandbox isolation layers with focus on identifying real-world bypass conditions under complex execution flows and weak architectural assumptions.
Reverse Engineering - Cross architecture binary analysis across ARM64, x86, and x64 systems spanning Windows, Linux, and Android including static analysis, dynamic tracing, and behavioral reconstruction of closed-source components.
Fuzzing & Automation - Design of targeted fuzzing systems including structured input generation, crash triage pipelines, exploitability validation, and custom harness development for both kernel and user space targets.
Source Code Audit - Large scale codebase analysis focused on architectural flaws, unsafe design patterns, and multi-stage vulnerability chains emerging across system boundaries rather than isolated implementation bugs.

Research Approach

  • Exploitability first reasoning applied across all research domains
  • Architecture level vulnerability discovery beyond implementation level bugs
  • Combination of logic flaws and memory corruption for multi stage exploitation paths
  • Cross boundary privilege escalation modeling across browser, OS, and kernel layers
